# SD Auto Generation Tool - 要件定義

## 概要
Stable Diffusion WebUIのAPIモードを利用した自動画像生成ツールを改良する。コマンドラインインターフェースでSDの起動確認、モード選択、キャラクター・シチュエーション選択、生成条件設定を行い、画像を日付別フォルダに出力する。

## 主要機能

### 1. SD APIモード起動確認
- プログラム起動時にSD WebUIのAPI（http://127.0.0.1:7860）が利用可能か確認
- 接続できない場合はエラーメッセージを表示して終了
- ユーザーは事前にSD WebUIをAPIモードで起動しておく

### 2. モード選択
- **生成モード**: 画像生成を実行
- **設定モード**: SD生成パラメータの設定変更
  - 設定項目: negative_prompt, steps, cfg_scale, width, height, sampler_name など
  - 設定は.envファイル（環境設定）とconfig/sd_config.txt（SDパラメータ）に保存

### 3. 生成モード詳細

#### キャラクター選択
- `src/character.txt` を読み込み（JSON配列形式）
- 各キャラクター: `{"en": "SD用プロンプト", "ja": "表示用日本語"}`
- コマンドラインに日本語名で一覧表示
- ユーザーが番号でキャラクターを選択（-1でランダム）

#### シチュエーション選択
- `situation/` フォルダ内のサブフォルダを一覧表示
- 各サブフォルダはシチュエーションカテゴリ（例: ファイルA, ファイルB）
- 各サブフォルダ内には `01_シチュ名.txt`, `02_シチュ名.txt` などのプロンプトファイル
- 各.txtファイルはプロンプトのJSON配列（`[{"en": "...", "ja": "..."}]`）

#### 生成設定
- 選択したシチュエーションごとに繰り返し回数を入力
- 選択したシチュエーションフォルダ全体の繰り返し回数を入力
- 生成条件の確認表示

#### 出力仕様
- 出力フォルダ: `output/YYYY-MM-DD/`
- ファイル名: `{キャラ名(ja)}_{シチュ名}_{連番}.png`
- 連番はキャラ・シチュごとに独立

### 4. 進捗表示とキャンセル
- 生成中は「現在 X/Y 個生成中 - キャラ: A, シチュ: B」のような表示
- Ctrl+C で中断可能
- 中断時は現在の生成をキャンセルし、SD APIにもinterruptリクエスト送信

## 全体システムの動作説明

このシステムはGo言語で実装されたStable Diffusion自動画像生成ツールです。以下の流れで動作します：

1. **初期化フェーズ**:
   - SD WebUI API (http://127.0.0.1:7860) の接続を確認
   - .envファイルとconfig/sd_config.txtから設定を読み込み
   - 必要なフォルダ構造を確認・作成

2. **モード選択フェーズ**:
   - ユーザーに生成モードまたは設定モードを選択させる
   - 生成モード: 画像生成を実行
   - 設定モード: SDパラメータの変更

3. **生成モード実行**:
   - `src/character.txt` からキャラクター一覧を読み込み表示
   - ユーザーがキャラクターを選択（またはランダム）
   - `situation/` フォルダ内のシチュエーションカテゴリを表示
   - 各シチュエーションの繰り返し回数と全体繰り返し回数を入力
   - 生成条件を確認

4. **画像生成フェーズ**:
   - 選択されたキャラクターとシチュエーションの組み合わせでプロンプト生成
   - SD APIにリクエスト送信
   - 生成された画像を `output/YYYY-MM-DD/` に保存
   - 進捗を表示し、Ctrl+Cでキャンセル可能

### 主要コンポーネントの役割

- **main.go**: エントリーポイント、全体の制御フロー
- **config/**: 設定ファイルの読み書き
- **ui/**: ユーザーインターフェース（入力、メニュー表示）
- **prompt/**: プロンプトデータの読み込み（character.txt, situationファイル）
- **situation/**: シチュエーションデータの管理
- **stableDiffusion/**: SD APIとの通信、画像生成・保存
- **common/**: 共通データ構造体
- **utils/**: ユーティリティ関数（日付処理、ファイル操作など）

### データフロー

```
ユーザー入力 → UI処理 → データ読み込み → プロンプト生成 → SD API → 画像保存
                                                            ↓
                                                  進捗表示・キャンセル処理
```

### エラーハンドリング

- SD API接続エラー: プログラム終了
- ファイル読み込みエラー: エラーメッセージ表示、該当機能スキップ
- ユーザー入力エラー: 再入力を促す
- 生成中断: 現在のリクエストをキャンセル、SD APIにもinterrupt送信
